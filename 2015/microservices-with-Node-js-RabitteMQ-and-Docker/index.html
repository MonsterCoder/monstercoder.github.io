Your start-up project with Node.js, RabbitMQ and Microservices

  • Node.js
  • architecture
  • container
  • microservice

These years I have worked with several projects for non-profite organizations and Startups. There are something in common to these projects.

  • Scalability: it starts from small, but might scale up radically
  • Agility: be able to move to different platforms
  • Flexiblity: components and features might be expanded or replaced
  • Management & Operations: don’t want to keep own infrastructure
  • Limited budget

Rails and Heroku are used to be my first choice for this kind of projects. Ruby on Rails is an amazing framework with a strong ecosystem. It is meant for speed and simplicty. While Heroku offers instant deployment, easy scaling and requires little manitainace.
